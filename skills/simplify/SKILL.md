# Code Simplifier

审查当前会话中修改或编写的代码，在保持功能不变的前提下简化和优化。

## 触发场景

- 用户要求简化/优化/重构代码
- 用户写完代码后要求 review
- 用户说 "simplify"、"clean up"、"优化一下" 等

## 核心原则

1. **功能不变**：只改写法，不改行为。所有原有功能、输出、行为必须保持不变。

2. **遵循项目规范**：如果项目有 CLAUDE.md 或类似规范文件，按规范来。没有的话按语言社区通用最佳实践。

3. **提升清晰度**：
   - 减少不必要的复杂度和嵌套
   - 消除冗余代码和过度抽象
   - 改善变量和函数命名
   - 合并相关逻辑
   - 去掉描述显而易见内容的注释
   - 避免嵌套三元运算符 — 用 if/else 或 switch
   - **清晰优于紧凑** — 明确的代码优于过度精简的代码

4. **保持平衡**，避免过度简化导致：
   - 降低可读性或可维护性
   - 产生"聪明但难懂"的写法
   - 把太多职责塞进一个函数
   - 移除有意义的抽象
   - 为了少写几行而牺牲可读性
   - 让代码更难调试或扩展

5. **聚焦范围**：只审查当前会话中修改或编写的代码，除非用户明确要求更大范围。

## 执行流程

1. 找到当前会话中最近修改/创建的代码文件
2. 分析可以简化和优化的地方
3. 应用项目规范和语言最佳实践
4. 确认所有功能保持不变
5. 直接应用修改（不只是建议）
6. 简要说明做了哪些关键改动及原因

## 输出要求

- 直接修改文件，不要只给建议
- 改完后简要列出关键改动点（3-5 条）
- 如果没有需要优化的地方，告诉用户代码已经很好，不需要强行修改
