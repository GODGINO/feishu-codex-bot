# 飞书工具问题排查

## ❓ 常见问题（FAQ）

### MCP 工具调用失败

**现象**：调用飞书 MCP 工具时返回错误或超时。

**排查步骤**：

1. **检查 MCP 绑定状态**：用户是否已绑定 Feishu MCP URL？
   - 如未绑定，引导用户参考 feishu-mcp-bind skill 完成绑定
2. **检查 MCP 连通性**：MCP URL 是否可访问？
   - 用户的 MCP URL 可能已过期，需要重新获取
3. **检查权限**：用户是否有对应资源的访问权限？
   - 文档/表格/日历等需要用户本人有权限

### 群聊中使用了错误的 MCP

**现象**：在群聊中调用飞书工具操作了错误用户的数据。

**原因**：群聊中每个用户有独立的 Feishu MCP，必须使用当前消息发送者对应的 MCP。

**解决**：
- 检查工具名前缀：`mcp__feishu_ou_<user_open_id>__<tool>`
- 确保 `<user_open_id>` 是当前发送消息的用户

### 卡片按钮点击无反应

**现象**：点击卡片按钮后没有任何反应。

**原因**：应用未开通「消息卡片回传交互」权限。

**解决步骤**：
1. 登录飞书开放平台：https://open.feishu.cn/app
2. 选择应用 → **事件与回调**
3. 添加回调 "卡片回传交互"(card.action.trigger)
4. 创建应用版本 → 提交审核 → 发布

### 文档/表格操作权限不足

**现象**：调用 create-doc、bitable 等工具时报权限错误。

**原因**：用户的 Feishu MCP 授权范围可能不包含所需权限。

**解决**：
1. 用户重新访问 `https://open.feishu.cn/page/mcp` 获取 MCP URL
2. 在授权页面确认已勾选所需权限范围
3. 重新绑定新的 MCP URL

### 日历/任务操作失败

**现象**：创建日程或任务时报错。

**常见原因**：
- 时间格式不正确（日历需要 ISO 8601，任务也是）
- 未传 current_user_id / user_open_id
- 人员 ID 格式不匹配

**解决**：参考 feishu-calendar-enhanced 和 feishu-task skill 中的参数说明。

## 🔍 通用诊断流程

1. **确认 MCP 绑定**：检查用户是否有 Feishu MCP
2. **检查工具名**：确认使用了正确的 MCP 前缀
3. **查看错误信息**：大多数飞书 API 错误会返回明确的错误码和消息
4. **重试**：部分错误可能是临时性的（网络、限流），可以重试
5. **权限升级**：如果是权限不足，引导用户重新授权
